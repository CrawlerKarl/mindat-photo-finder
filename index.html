Why there are two searches

Primary matches Mindat's structured species list for the photo — reliable, but only one mineral can go here.
Associate matches the caption text, so it only finds photos where the photographer actually wrote that mineral's name.

Because the two fields search different things, running it both ways and taking the union is the honest version of "A with B" — that's what Open both does. It's also why a third mineral needs its own query: the caption field matches a phrase, not a set of keywords.
