Why there are two searches

Primary matches Mindat's structured species list for the photo — reliable, but only one mineral can go here.
Associate matches the caption text, so it only finds photos where the photographer actually wrote that mineral's name.

Because the two fields search different things, running it both ways and taking the union is the honest version of "A with B" — that's what Open both does. It's also why a third mineral needs its own query: the caption field matches a phrase, not a set of keywords.

Starting points

Three or more minerals

Mindat has no AND search, so pages like this one can't intersect three minerals for you — but a bookmarklet running on mindat.org itself can. It sweeps each mineral as a caption term, verifies every result row against Mindat's structured species list, and shows only photos carrying all of your minerals — with live progress, pause, and CSV export. Polite by design: sequential page views in your own browser session, ~0.8 s apart, hard-capped.

Or drag this button to the bookmarks bar: ◆ Mineral Finder

Clicked it and the URL changed instead of a panel opening? Diagnose it here
  • URL became this page → the drag saved the page, not the button. Delete that bookmark and use the copy-paste install above.
  • URL became a mindat “File not found” page → your browser stripped the javascript: prefix when saving. Right-click the bookmark → Edit → the URL must begin with javascript:(function() — if it doesn't, re-paste the code (and if the prefix vanishes on paste, type javascript: in front by hand).
  • Nothing at all → make sure you're on a normal mindat.org page (not the Cloudflare “checking your browser” interstitial), then click again.
  • Using the DuckDuckGo browser → it doesn't execute javascript: bookmarks at all (it navigates to them as addresses), and it has no extension support to substitute. Use Chrome, Safari, Firefox, or Edge for the finder — DuckDuckGo remains fine for everything else on mindat.

Works in: Chrome, Edge, Firefox, Safari (desktop). Doesn't work in: DuckDuckGo browser and most mobile in-app browsers — they refuse javascript: bookmarks by design.

On a phone

Mobile browsers have no bookmarks bar, but three routes work. The panel itself is phone-sized.

iPhone / iPad — Safari (no installs, ~1 min)
  1. On this page, tap Copy the bookmarklet code above
  2. Bookmark any page (Share → Add Bookmark), then Bookmarks → Edit → open that bookmark, rename it Mineral Finder and replace its address with the pasted code
  3. On any mindat.org page: Bookmarks → tap Mineral Finder — the panel opens in place
Android — Chrome (the address-bar trick)
  1. Add the bookmark with the pasted code as its URL (or let it sync from desktop Chrome), named something short like mmf
  2. On a mindat.org page, tap the address bar, type mmf, and tap the bookmark in the suggestions. This is the trick: launching it from the bookmarks menu navigates instead of running — only the address-bar suggestion executes it on the current page
Userscript — smoothest for repeat use (Firefox Android + Tampermonkey, iOS “Userscripts” extension)

Install intersect.user.js in your userscript manager. Every mindat.org page then shows a floating ◆ Finder button — tap it to open the panel. No bookmarklet needed.

Completeness, honestly stated: a photo is findable only if its caption mentions at least one of your minerals — the final species check is exact, the discovery step depends on captions. The panel repeats this on every finished run.
